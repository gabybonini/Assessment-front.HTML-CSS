Design Responsivo

  1. Quando é recomendado a criação de um site separado para mobile? Seria este o caso deste site da Câmara dos Deputados?

    Resposta: Sites para mobile são rapidos devido a sua implementação ser mais específica o que torna seus arquivos menores. A criação para eles é recomendada quando a performace é prioridade. Pelo tamanho do site que criamos não há necessidade.

  2. No caso de um CSS específico para impressão (print), como ele deve ser inserido?

    Resposta: Pode se criar uma folha de estilos separada e incorporá-la ao HTML adicionando a propriedade na incorporação. Ou utilizando folhas de estilos já existentes

multimídia Web

  1. O elemento audio permite como elemento-filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento audio e quanto devem ser utilizados?

    Resposta: Atributos do elemento f.

    Atributo Valor Descrição
    media media query Aceita qualquer media query que poderia ser definido no CSS
    size Especifica o tamanho da imagem para diferentes layouts de pagina
    src URL Especifica a URL do arquivo de mídia. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    srcset URL Especifica a URL da imagem que será utilizada em diferentes situações. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    type MIME-type Especifica o MIME-type do recurso
    Atributo Descrição
    autoplay Especifica que o audio vai começar a tocar assim que estiver pronto
    controls Especifica que os controles de audio devem ser mostrados (play, pause, etc)
    loop Especifica que o audio vai começar novamente, cada vez que terminar
    muted Especifica que a saída de audio deve ser silenciada
    preload Especifica se e como o audio deve ser carregado quando a pagina carregar
    src Especifica a URL do arquivo de audio

  2. O elemento vídeo permite como elemento-filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento vídeo e quando devem ser utilizados?

    Atributo Valor Descrição
    media media query Aceita qualquer media query que poderia ser definido no CSS
    size Especifica o tamanho da imagem para diferentes layouts de pagina
    src URL Especifica a URL do arquivo de mídia. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    srcset URL Especifica a URL da imagem que será utilizada em diferentes situações. Esse atributo é obrigatório quando a tag source é utilizada nas tags video e audio.
    type MIME-type Especifica o mime-type do recurso
    Atributo Descrição
    autoplay O video vai começar assim que estiver disponível
    controls Os controles de vídeo devem ser mostrados (plat, pause, etc)
    height Define a altura do player
    loop O vídeo vai começar novamente, toda vez que terminar.
    muted A saída de audio do video deve ser silenciada
    poster Uma imagem a ser exibida enquando o vídeo está sendo baixado ou até que o usuario clique no botão play
    preload O video deve ser carregado quando a pagina carregar
    src A URL do arquivo de vídeo
    width Define a largura do player

  3. No caso do canvas, HTML5 nunca, em momento algum, fez qualquer animação. Isso porque o HTML5 é uma simples linguagem de marcação. Dessa forma, o que é necessário para criar animações com HTML5? O que são APIs do HTML5?

    Resposta: O canvas HTML é uma API com a qual podemos interagir através de códigos javascript. São instruções disponíveis nos navegadores que facilitam o acesso a funcionalidades complexas através da abstração de código.

  4. Como o CSS3 pode ser aplicado na aparência da multimídia HTML5. Demonstre com um exemplo.

    Resposta: Podemos utilizar o CSS3 para ajustar as dimensões do player de video, e tamanho do video a ser exibido.
